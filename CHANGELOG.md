# 版本历史

## v4.0.0 · 2026-06-25 · 智能化可视化增强统计版

**核心理念**：在保持零干扰原则的前提下，实现智能化、可视化、增强统计三大能力升级。

### 数据层重构

- 新增 SQLite 数据库模块（`mcp/database.mjs`）：统一数据存储层
- 新增日志轮转模块（`mcp/log-rotation.mjs`）：自动轮转和压缩日志
- 审计日志结构化增强：增加 session_id、model、turn、text_hash 字段
- 多日志分级：`audit-full.log`、`audit-violations.log`、`audit-summary.log`
- 审计日志自动触发：后台每 30 秒自动检查并记录

### 统计引擎

- 新增会话追踪模块：每次 OpenCode 启动生成唯一会话 ID
- 新增时间维度聚合：按小时/天/周/月统计
- 新增违规分类统计：按类型（英文句子/填充词/混合短语/代码注释）分布
- 新增热力图数据：7×24 违规热力图
- 新增仪表板数据接口（`zhongwen_dashboard` 工具）

### 可视化层

- 新增 Web 仪表盘服务器（`mcp/dashboard.mjs`）
- 新增实时监控面板：显示当前状态、纯度、违规次数
- 新增趋势图表：纯度和违规率趋势线图（Chart.js）
- 新增违规分布图：饼图/环形图
- 新增时间热力图：GitHub 风格贡献图
- 新增仪表盘 MCP 工具（`zhongwen_open_dashboard`）
- 新增前端页面（`docs/dashboard/index.html` + `app.js`）

### 智能化引擎

- 新增用户行为模式分析（`mcp/intelligence.mjs`）
- 新增自适应白名单建议
- 新增智能告警：连续 FAIL、违规率过高、纯度骤降
- 新增会话质量评分：0-100 分加权模型
- 新增违规根因分析：自动识别违规模式并生成改进建议

### 架构变更

- 数据层统一：SQLite 替代散落的日志文件
- 可视化双渠道：命令行 + Web 仪表盘
- 智能化：模式识别、自适应学习、智能告警
- 完全向后兼容：所有现有功能保持不变

## v3.0.0 · 2026-06-24 · 零干扰强约束重构版

**核心理念**：约束施加在不可控层，零要求施加在可控层。

### 核心变更

- 重构 Agent 定义文件（178 行 → 7 行）：纯身份定义，零强制条款
- 重构全局指令文件（69 行 → 25 行）：身份定义 + 中文化表达 + 可选自检
- 从"前台强制"转为"后台监控 + 可选自检"
- 智能体正常工作流程零改变（无格式锁、无强制调用、无强制重写）

### 检测增强

- 新增混合短语检测（中英字符交替出现 ≥3 次）
- 新增代码块注释检查（`//`、`#`、`/*` 等注释行）
- 新增白名单/豁免机制（`~/.config/opencode/whitelist.json`）
- 新增修复建议映射（英文 → 中文自动翻译建议）
- 新增审计日志自动记录（`language-audit.log`）

### 架构变更

- 约束全在不可控层：系统提示身份定义 + MCP 外部进程
- 可控层零要求：不要求调用工具、不要求改输出格式、不要求任何额外操作

### 版本管理

- 新增 `zhongwen_status`、`zhongwen_snapshot`、`zhongwen_rollback`、`zhongwen_upgrade`、`zhongwen_history` MCP 工具
- 智能体可直接在对话中调用版本管理命令
