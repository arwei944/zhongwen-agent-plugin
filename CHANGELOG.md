## v4.8.0 · 2026-06-27 · 压缩后约束恢复版

**核心理念**：解决会话压缩后约束失效的问题，确保压缩后约束依然有效。

### 核心变更

- 在系统提示词中新增"压缩后约束恢复协议"条款
- 明确压缩后识别信号和强制恢复动作
- MCP 检查器新增压缩恢复提醒机制
- 当检测到连续 2 次以上 BLOCKED 或纯度骤降时，自动附加压缩恢复提醒
- get_session_stats 工具也支持压缩恢复提醒
- 版本号统一升级至 v4.8.0

## v4.7.0 · 2026-06-27 · 锚定文本豁免版

**核心理念**：解决 think 块标准中文锚定文本反复触发语言检查导致阻断和重试的问题。

### 核心变更

- 语言检查器新增 think 块锚定文本识别与跳过逻辑
- 以“【”开头的中文锚定行，若包含“中文思维已激活”、“语言纯度承诺”、“当前任务”、“检查承诺”、“禁止声明”等关键词，直接跳过英文检测
- 避免标准中文锚定文本被误判为违规，减少不必要的阻断和重试
- 节省 token 消耗和响应时间
- 版本号统一升级至 v4.7.0

## v4.5.0 · 2026-06-27 · 思维格式简化版

**核心理念**：将 think 块锚定格式从 5 行简化为 3 行，保持 100% 中文严格程度，提升折叠友好性。

### 核心变更

- think 块格式从 5 行压缩为 3 行：
  - 第 1 行：（【中文思维已激活】本段思考严格使用中文，任何英文自然语言都是违规。）
  - 第 2 行：（语言纯度承诺：我在本段思考中将保持 100% 中文。）
  - 第 3 行：（当前任务：[用中文一句话概括用户请求]）
- 统一主仓库与插件仓库的 think 块格式
- 保持 100% 中文严格程度，未降低任何约束
- 版本号统一升级至 v4.5.0

## v4.0.0 · 2026-06-26 · 工程级强约束版本（6道不可绕过工程锁）

**核心理念**：零信任架构 + 不可绕过约束 + 渐进式惩戒。

### 核心变更

- 全新 Agent 定义（`zhongwen-agent.md`）：6 道工程锁 + 渐进式惩戒协议
- 全新全局指令（`chinese-rules.md`）：强制门卫检查 + 输出标记 + 禁止行为
- MCP 语言检查器升级为 v4.0.0：实时双向检查 + 自动修复引擎 + 会话级状态机

### 6 道不可绕过工程锁

1. **外部独立进程门卫**：`check_chinese_purity` 工具强制调用，AI 无法 bypass
2. **思维格式硬锚定**：think 块前 5 行固定格式，不可变更
3. **语言纯度实时检查**：双向检查（think + output），实时计算中文纯度
4. **词汇与句式白名单**：只允许使用中文逻辑连接词，禁止英文等价物
5. **输出格式锁定**：纯中文段落 + 术语解释 + 检查标记
6. **违规惩戒与自省**：渐进式惩戒（重写→反省→会话重置→人工介入）

### MCP 检查器增强

- 会话级状态机：跟踪违规次数，4 次违规自动锁定
- 自动修复引擎：英文→中文自动替换建议
- 实时 think 块检查：检测思考内容中的英文违规
- 输出拦截：检查失败直接阻断输出，AI 无法绕过

### 架构变更

- 删除日志系统：移除 database.mjs、log-rotation.mjs、dashboard.mjs、intelligence.mjs
- 删除所有文件日志和审计日志
- 移除后台检查器、告警系统、仪表盘等非核心功能
- 核心检查器精简为纯语言检查 + 会话状态机
- 约束全在不可控层：系统提示 + 外部 MCP 进程
