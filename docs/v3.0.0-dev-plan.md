# v3.0.0 开发文档 · 零干扰强约束重构版

> **版本目标**：实现"最强约束 + 完全不影响智能体正常工作"的双核心诉求

---

# 开发总览

## 版本代号

**v3.0.0 · 零干扰强约束重构版**

## 核心理念

```
约束 = 施加在「智能体不可控的层面」
干扰 = 发生在「智能体可控的层面」

最强约束 + 零干扰 = 约束全在不可控层，可控层零要求
```

## 改动范围

| 文件 | 当前版本 | v3.0.0 目标 | 改动性质 |
|:---|:---|:---|:---:|
| `agents/zhongwen-agent.md` | 178 行 | ~12 行 | 重构 |
| `chinese-rules.md` | 69 行 | ~8 行 | 重构 |
| `mcp/check_language.mjs` | 13.6 KB | ~18 KB | 增强 |
| `mcp/manage.mjs` | 14.2 KB | ~14.2 KB | 无改动 |
| `opencode.json` | 1.8 KB | 微调 | 小改 |
| `README.md` | 6.5 KB | ~8 KB | 更新 |
| `CHANGELOG.md` | 1.1 KB | ~1.5 KB | 更新 |

## 验收总标准

1. 智能体输出格式**零改变**（无格式锁、无附加内容、无强制工具调用）
2. 智能体思考内容**零要求**（无锚定格式、无强制自检、无强制重写）
3. MCP 语言检查服务器**自动后台运行**（不依赖智能体调用）
4. 所有检测能力**后台记录**到审计日志（不中断工作流）
5. 回滚/升级命令**100% 可用**（v2.2.0 的 MCP 版本管理工具保持不变）
6. 代码**通过 Node.js 语法检查**（`node --check` 无错误）

## 任务 3：重构 MCP 语言检查服务器

### 3.1 文件变更

**文件**：`mcp/check_language.mjs`

**当前问题**：
- 仅提供 3 个工具（需要智能体主动调用）
- 无后台自动检查能力
- 无审计日志写入
- 无混合短语检测
- 无代码块注释检查
- 无白名单机制
- 无修复建议

**目标状态**：

#### 3.1.1 新增后台审计日志

在 `analyzePurity()` 调用路径中，自动将结果写入审计日志：

```javascript
const AUDIT_LOG = 'C:\\Users\\Administrator\\.config\\opencode\\logs\\language-audit.log';

function writeAuditLog(report, context = 'unknown') {
  const entry = {
    timestamp: new Date().toISOString(),
    context,  // 'background' | 'self-check' | 'manual'
    status: report.status,
    purity: report.purity,
    violations: report.violations.length,
    details: report.details
  };
  // 写入日志
}
```

**验收标准**：
- 每次 `check_chinese_purity` 工具被调用时，自动写入审计日志
- 日志格式为 JSON Lines（每行一个 JSON 对象）
- 日志文件路径：`~/.config/opencode/logs/language-audit.log`
- 写入不阻塞工具响应（异步写入）

#### 3.1.2 新增混合短语检测

在 `analyzePurity()` 函数中增加中英字符过渡检测：

```javascript
function detectMixedPhrases(line) {
  const transitions = [];
  let lastType = null;
  for (const ch of line) {
    const code = ch.codePointAt(0);
    const type = isChineseChar(code) ? 'zh' : 
                 isEnglishChar(code) ? 'en' : null;
    if (type && lastType && type !== lastType) {
      transitions.push({ from: lastType, to: type, char: ch });
    }
    lastType = type;
  }
  return transitions;
}
```

**验收标准**：
- 输入 `"这个function处理data然后return结果"` 检测到 ≥ 3 次过渡
- 输入 `"使用JWT进行身份认证"` 检测到 0 次过渡（JWT 在大写缩写白名单中）
- 输入 `"let me analyze this"` 不触发混合检测（已被英文句子检测捕获）

#### 3.1.3 新增代码块注释检查

修改代码块跳过逻辑，对注释行进行语言检查：

```javascript
if (inCodeBlock && options.codeBlockMode) {
  // 检查注释行
  const trimmed = line.trimStart();
  if (/^(\/\/|#|;|\/\*|\*|<!--|--)/.test(trimmed)) {
    checkCommentLanguage(line, i + 1, report);
  }
  continue;
}
```

**验收标准**：
- 代码块中的 `// 初始化用户列表` → 通过（中文注释）
- 代码块中的 `// init user list` → 违规（英文注释）
- 代码块中的 `// 使用 bcrypt 加密` → 通过（中文注释，bcrypt 在白名单中）
- 代码块中的 `function calculate() {}` → 跳过（非注释）

#### 3.1.4 新增白名单机制

新增 `whitelist.json` 配置文件：

```json
{
  "allowed_terms": ["JWT", "API", "REST", "HTTP", "HTTPS", "URL", "JSON", "SQL"],
  "allowed_patterns": ["^[A-Z]{2,8}$", "^[a-z]+\\.[a-z]+$"],
  "skip_code_blocks": true,
  "check_comments": true,
  "thresholds": {
    "purity_pass": 90,
    "purity_warn": 70
  }
}
```

**验收标准**：
- `whitelist.json` 文件存在且可被 MCP 服务器加载
- 白名单中的术语不触发违规告警
- 白名单支持正则表达式模式
- 阈值可配置

#### 3.1.5 新增修复建议

当检测到违规时，返回修复建议：

```javascript
const FIX_SUGGESTIONS = {
  'let me': '让我',
  'i need to': '我需要',
  'first of all': '首先',
  'edge case': '边界情况',
  'fallback': '回退机制',
  'approach': '方法',
  'scenario': '场景',
  'actually': '实际上',
  'as a result': '因此',
  'in other words': '也就是说'
};
```

**验收标准**：
- 违规项包含 `suggestion` 字段
- 建议为中文翻译
- 无匹配时 suggestion 为空字符串

#### 3.1.6 保持现有工具接口

**验收标准**：
- `check_chinese_purity(text, codeBlockMode)` 接口不变
- `get_session_stats()` 接口不变
- `reset_session_stats(confirm)` 接口不变
- 新增的检测能力不破坏现有接口
