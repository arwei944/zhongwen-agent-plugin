### 任务 M1-03：审计日志自动触发机制

#### 任务描述

在零干扰模式下，MCP 服务器定期（默认 30 秒）对最近的输出进行后台检查并写入审计日志，不依赖 AI 主动调用工具。

#### 输入

- 现有的 `sessionTracker` 对象
- OpenCode 的输出上下文（通过 MCP 通知获取）

#### 输出

- 新增的 `backgroundChecker` 模块
- 配置化的检查间隔（`check_interval`，默认 30000ms）
- 最近输出的缓存机制

#### 依赖关系

- 依赖 M1-01（数据库模块）
- 依赖 M1-02（结构化日志）

#### 验收标准

| 检查项 | 通过标准 | 测试方法 |
|:---|:---|:---|
| 自动触发 | 不调用 check_chinese_purity 工具时，日志仍增加 | 等待 35 秒后检查日志条数 |
| 检查间隔 | 两次检查间隔 ≈ 30 秒（允许 ±5 秒误差） | 对比两条日志的时间戳 |
| 不影响 AI 输出 | 自动检查期间 AI 响应时间无显著增加 | 对比有无自动检查的响应时间 |
| 可配置 | 支持修改检查间隔 | 修改配置后验证新间隔生效 |
| 可禁用 | 支持关闭自动检查 | 设置 interval=0 后验证无新日志 |

#### 验证命令

```powershell
# 记录当前日志行数
$before = (Get-Content "$env:USERPROFILE\.config\opencode\logs\language-audit.log").Count

# 等待 35 秒
Start-Sleep -Seconds 35

# 检查日志是否增加
$after = (Get-Content "$env:USERPROFILE\.config\opencode\logs\language-audit.log").Count
$after -gt $before
```

### 任务 M1-04：多日志分级

#### 任务描述

将单一审计日志拆分为三级：全量日志、违规日志、汇总日志。

#### 输入

- 现有的 `writeAuditLog` 函数
- 现有的 `AUDIT_LOG_PATH` 配置

#### 输出

- 三个日志文件：
  - `audit-full.log` — 全量检查记录
  - `audit-violations.log` — 仅违规记录
  - `audit-summary.log` — 每会话汇总（会话结束时写入）

#### 依赖关系

- 依赖 M1-01（数据库模块）
- 依赖 M1-02（结构化日志）

#### 验收标准

| 检查项 | 通过标准 | 测试方法 |
|:---|:---|:---|
| 全量日志存在 | `audit-full.log` 包含所有检查 | 执行 10 次检查，验证全量日志有 10 条 |
| 违规日志存在 | `audit-violations.log` 仅包含违规 | 执行 10 次检查（5 次 PASS，5 次 FAIL），验证违规日志有 5 条 |
| 汇总日志存在 | `audit-summary.log` 每会话一条 | 启动两个会话，验证有两条汇总 |
| 汇总字段完整 | 包含会话统计（total_checks、violations、avg_purity） | 解析汇总日志验证字段 |
| 日志互不干扰 | 写入一个日志不影响其他日志 | 并发写入验证 |

#### 验证命令

```powershell
# 验证三个日志文件都存在
$files = @(
    "$env:USERPROFILE\.config\opencode\logs\audit-full.log",
    "$env:USERPROFILE\.config\opencode\logs\audit-violations.log",
    "$env:USERPROFILE\.config\opencode\logs\audit-summary.log"
)
foreach ($f in $files) { Test-Path $f }

# 验证违规日志仅包含 FAIL 记录
$violations = Get-Content "$env:USERPROFILE\.config\opencode\logs\audit-violations.log" | ForEach-Object { $_ | ConvertFrom-Json }
$violations | ForEach-Object { $_.status -eq "FAIL" } | Where-Object { $_ -eq $false } | Measure-Object | Select-Object -ExpandProperty Count
# 应为 0
```

# 附录：数据库 Schema

```sql
-- 会话表
CREATE TABLE sessions (
    session_id TEXT PRIMARY KEY,
    start_time TEXT NOT NULL,
    end_time TEXT,
    model TEXT,
    agent TEXT,
    pid INTEGER,
    total_checks INTEGER DEFAULT 0,
    total_violations INTEGER DEFAULT 0,
    avg_purity REAL DEFAULT 100.0,
    quality_score REAL DEFAULT 100.0,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP
);

-- 检查记录表
CREATE TABLE checks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT NOT NULL,
    timestamp TEXT NOT NULL,
    purity REAL NOT NULL,
    status TEXT NOT NULL,  -- PASS/WARN/FAIL
    chinese_chars INTEGER DEFAULT 0,
    english_chars INTEGER DEFAULT 0,
    total_chars INTEGER DEFAULT 0,
    text_hash TEXT,
    context TEXT,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (session_id) REFERENCES sessions(session_id)
);

-- 违规记录表
CREATE TABLE violations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    check_id INTEGER NOT NULL,
    session_id TEXT NOT NULL,
    timestamp TEXT NOT NULL,
    violation_type TEXT NOT NULL,  -- english_sentence/english_filler/mixed_phrase/comment
    line_number INTEGER,
    content TEXT,
    suggestion TEXT,
    context TEXT,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (check_id) REFERENCES checks(id),
    FOREIGN KEY (session_id) REFERENCES sessions(session_id)
);

-- 版本快照表
CREATE TABLE versions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    version TEXT NOT NULL,
    timestamp TEXT NOT NULL,
    type TEXT DEFAULT 'snapshot',
    files_count INTEGER DEFAULT 0,
    metadata TEXT,  -- JSON
    created_at TEXT DEFAULT CURRENT_TIMESTAMP
);

-- 告警记录表
CREATE TABLE alerts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT,
    alert_type TEXT NOT NULL,
    severity TEXT NOT NULL,  -- INFO/WARNING/CRITICAL
    message TEXT NOT NULL,
    data TEXT,  -- JSON
    acknowledged BOOLEAN DEFAULT 0,
    timestamp TEXT DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (session_id) REFERENCES sessions(session_id)
);

-- 索引
CREATE INDEX idx_checks_session ON checks(session_id);
CREATE INDEX idx_checks_timestamp ON checks(timestamp);
CREATE INDEX idx_checks_status ON checks(status);
CREATE INDEX idx_violations_check ON violations(check_id);
CREATE INDEX idx_violations_session ON violations(session_id);
CREATE INDEX idx_violations_type ON violations(violation_type);
CREATE INDEX idx_violations_timestamp ON violations(timestamp);
CREATE INDEX idx_sessions_model ON sessions(model);
CREATE INDEX idx_alerts_timestamp ON alerts(timestamp);
CREATE INDEX idx_alerts_type ON alerts(alert_type);
```
